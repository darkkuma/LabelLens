# LabelLens: 3-Day Submission Sprint

## North Star

In under 60 seconds, a Korean shopper searches or scans a packaged food, understands the
tradeoffs, changes one personal priority, and finds a better product in the same category.

## Day 1: Product Foundation

- Korean-first responsive redesign
- Explainable five-part score
- Personal priorities that change score and ranking
- MFDS nutrition search and C002 ingredient matching
- Korean label parser with origin-boundary regression checks
- Desktop and mobile visual QA

## Day 2: Signature Experience

- Camera capture and Korean OCR for nutrition and ingredient panels
- Side-by-side product comparison
- Expand the additive evidence library with direct source links and review dates
- Add 20-30 verified products across dumplings, ramen, ready meals, and meal kits
- Add shareable analysis URLs and a concise data-confidence indicator

## Day 3: Submission

- Record a 60-90 second demo with one clear shopper story
- Capture desktop and mobile screenshots
- Write Devpost problem, solution, architecture, data, and responsible-health sections
- Document Codex usage and save the shared session ID
- Run accessibility, broken-link, API-fallback, and clean-browser tests
- Freeze features six hours before the deadline and publish the final build

## Judge Demo Script

1. Search `비비고 사찰만두`.
2. Show why 78 is not a black-box score.
3. Switch to `원산지 투명성` and show the score/rank respond.
4. Explain one additive without fear-based language.
5. Paste an unknown Korean label and parse it instantly.
6. Close on the public-data architecture and the camera/OCR vision.
